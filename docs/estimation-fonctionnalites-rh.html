03

Périmètre sprint

✓
Inclus dans ce sprint
  • Création et gestion du profil candidat
  • Entretiens multiples avec résultat (accepté / refusé)
  • Suivi de l'offre envoyée
  • Notification email HR à l'acceptation d'une offre
  • Création du profil complet après recrutement
  • Gestion des statuts : période d'essai → titulaire → démission
  • Notification automatique fin de période d'essai
  • Workflow démission / désactivation compte
  • Création et suivi de demandes avec pièces jointes
  • Interface RH d'approbation / rejet des demandes
→
Différé (sprint suivant)
  • Tableau de bord analytique recrutement (KPIs, taux)
  • Intégration calendrier entretiens (sync externe)
  • Scoring et comparaison candidats
  • Signature électronique de l'offre
  • Portail candidat (espace self-service)
  • Tests E2E automatisés (Playwright / Cypress)
  • Gestion avancée des congés / absences
  • Mobile responsive poussé
04

Hypothèses

⚙️
Stack technique
Spring Boot 4 / Java 17 / SQL Server (backend) + Angular 21 Signals standalone (frontend). Cohérence avec le projet DAF360 existant.
👤
Ressource
1 développeur full-stack sans interruption majeure. En mode 2 devs (BE + FE) le sprint passe à ~1 semaine calendaire.
📧
Emails
SMTP simple (JavaMailSender). Pas de service tiers (SendGrid, Mailjet). Templates HTML basiques.
📎
Stockage fichiers
Stockage local ou S3 compatible. Pas de gestion DRM ni de prévisualisation avancée en sprint 1.
🎨
Design
Cohérence avec le système de design DAF360 existant. Tokens CSS, composants partagés. Pas de nouvelles maquettes from scratch.
🧪
Tests
Tests unitaires inclus (JUnit 5, Jasmine/Jest). Pas de tests E2E automatisés en sprint 1. Tests manuels sur les golden paths.
