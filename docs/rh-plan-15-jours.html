Tâches planifiées

Module 1 — Recrutement

Jours 1–7 · Tasks 1–4
  • Création et gestion des profils candidats (CRUD)
  • Planification d'entretiens multiples (RH → Technique → Manager → Final)
  • Saisie des résultats d'entretien avec feedback
  • Envoi d'offre d'embauche avec date prévue et salaire
  • Flux acceptation / refus d'offre avec notification email RH
  • Frontend : liste candidats, formulaire création, fiche détail avec timeline

Module 2 — Cycle de vie employé

Jours 8–9 · Task 5
  • Statuts successifs : Période d'essai → Titulaire → Démission
  • Détection automatique de la fin de période d'essai (job planifié quotidien)
  • Notification email RH lors de la fin de période d'essai
  • Action de titularisation et de démission avec désactivation du compte

Module 3 — Gestion des demandes

Jours 10–13 · Tasks 6–7
  • Soumission de demandes par les employés (document, attestation, certificat…)
  • Suivi du statut : En attente → En cours → Approuvé / Refusé
  • Interface RH pour traitement des demandes avec notes de réponse
  • Vue historique des demandes par employé

Endpoints REST implémentés

Module 1 Recrutement

MéthodeURLRôleDescription
GET /api/hr/recruitment/candidates HR, ADMIN, MANAGER Liste complète des candidats
POST /api/hr/recruitment/candidates HR, ADMIN Créer un nouveau profil candidat
GET /api/hr/recruitment/candidates/{id} HR, ADMIN, MANAGER Détail candidat avec entretiens et offre
POST /api/hr/recruitment/interviews HR, ADMIN Planifier un entretien (round auto-incrémenté)
PUT /api/hr/recruitment/interviews/{id}/outcome HR, ADMIN, MANAGER Saisir le résultat (PASSED / FAILED) + feedback
POST /api/hr/recruitment/offers HR, ADMIN Envoyer une offre d'embauche
PUT /api/hr/recruitment/offers/{id}/accept HR, ADMIN Confirmer l'acceptation → notification email RH
PUT /api/hr/recruitment/offers/{id}/reject HR, ADMIN Enregistrer le refus avec motif

MéthodeURLRôleDescription

Risques & Mitigations

RisqueProbabilitéImpactMitigation
Conflit de colonnes JPA sur Employee (ddl-auto: update) Faible Moyen Vérifier le log de démarrage — colonnes ajoutées silencieusement via ALTER TABLE
Email SMTP Office365 bloqué en développement Moyenne Faible HrNotificationService attrape les exceptions — l'app démarre même si le mail échoue
CHECK CONSTRAINT sur la table RolePermissions Moyenne Moyen Prévoir un script SQL pour ajouter les nouvelles permissions avant le déploiement
Build Angular : imports manquants sur nouveaux composants Faible Faible Lancer npx ng build après chaque Task frontend — corriger immédiatement
Débordement sur J14–J15 Moyenne Moyen Module 3 (Demandes) est le premier candidat au report en V2 si le sprint prend du retard

Hypothèses

  • 2 développeurs disponibles à temps plein sur les 15 jours
  • Base de données SQL Server accessible en local (port 1433, base DAF360_HR)
  • JWT et cookies HttpOnly déjà fonctionnels (authentification Azure AD existante)
  • Email SMTP Office365 configuré dans application-local.yml (ou Mailtrap pour les tests)
  • Tables créées automatiquement par JPA (ddl-auto: update) — pas de Flyway nécessaire
  • Pas d'upload de fichiers dans ce sprint (reporté en V2)
  • Design cohérent avec l'existant DAF360 — pas de maquettes Figma requises
